
React Router

1. 라이브러리 설치
    - npm i react-router-dom@6 (6버전으로 설치)

2. Router 를 활용한 컴포넌트 이동
    - 기본 URL + "라우팅 정보"
        - "http://localhost:5173/" + "라우팅 정보"
        - "/" : home 페이지
        - "/new" : 새로운 일기 작성 페이지
        - "/diary" : 일기 목록 페이지

    - 프로젝트에 라우터 적용 : BrowserRouter
        - 브라우저의 주소 변경을 감지하는 기능이 있음
        - 컴포넌트가 페이지를 구성하고 이동하는 데 필요한 기능을 제공
        
        - App 의 부모 컴포넌트로 설정

    - 페이지에 라우팅 구현
        - <Routes> , <Route>
        - Routes 는 자신이 감싸는 Route 컴포넌트 중에서 브라우저 주소 표시줄에 입력된 
          URL 경로와 일치하는 요소를 찾아 페이지에 랜더링함 

3. Link 컴포넌트를 이용한 이동
    <Link to="이동할 경로">링크 이름</Link>

    to 속성에 경로를 지정해 페이지 이동
    React 는 클라이언트 사이드 랜더링 방식이므로 페이지를 서버에 요청하지 않고 브라우저가 직접 이동시킴
    즉, 브라우저에서 컴포넌트만 교체하는 방식으로 랜더링됨

    vs <a> 태그
    <a> 태그를 이용하면 브라우저는 현재 페이지를 지우고 새로운 페이지를 불러오는 방식으로 이동함
    -> 클라이언트 사이드 랜더링 방식 X
    -> 페이지 일부만 교체해 빠르게 이동하는 리액트 장점 활용 불가..

4. 동적 경로이동(라우팅)
    - 동적 경로
      특정 아이템을 나타내는 id 처럼 값이 변하는 요소를 URL에 포함하는 경우
      동적 경로를 표현하는 방법 2가지

      1. URL 파라미터
        - http://localhost:5173/diary/{id}
        - { } 를 이용해 URL 에 유동적인 값을 넣는 방법
        - <Route path="/diary/:id" element={<Diary />} />
     
      2. 쿼리 스트링
        - http://localhost:5173?sort=latest&page=1
        - ? 뒤에 key=value 문법으로 URL 에 유동적인 값 포함
